================================================================================
                         GENERAL GUIDELINES
================================================================================

1. PRESERVE EXISTING FUNCTIONALITY
   - Do not break existing features
   - Only modify styling, not logic
   - Test after changes

2. CONSISTENCY IS KEY
   - Use existing CSS variables (--primary, --background, etc.)
   - Follow the established spacing scale
   - Use Inter font consistently

3. MICRO-INTERACTIONS MATTER
   - Add subtle hover transitions (150-300ms)
   - Use transform: scale(0.98) for button presses
   - Add smooth focus transitions

4. DARK MODE MUST WORK
   - Test all changes in both light and dark mode
   - Use .dark class variants properly
   - Ensure sufficient contrast

5. MOBILE FIRST
   - Check responsiveness at 375px, 768px, 1024px widths
   - Touch targets should be at least 44px
   - Avoid hover-only interactions on mobile

6. CODE QUALITY
   - Keep Tailwind classes organized
   - Extract repeated patterns into index.css
   - Add meaningful comments for complex styles

================================================================================
                         HOW TO TEST CHANGES
================================================================================

1. Run the dev server:
   cd frontend
   npm run dev

2. Backend must be running:
   cd backend  
   python -m uvicorn main:app --reload

3. Test on:
   - Light mode
   - Dark mode (toggle in settings or system preference)
   - Different screen sizes

4. Pay attention to:
   - Console errors
   - Layout shifts
   - Flickering animations
